## 产品概述

Warp 是一个现代化的代理开发环境（Agentic Development Environment），专为帮助开发人员通过 AI 代理更快地交付代码而构建。与传统终端不同，Warp 集成了 AI 驱动的代码生成、智能命令建议和现代化文本编辑功能，将终端体验提升到 IDE 级别。

该产品使用 Rust 构建，支持 zsh、bash、fish 和 PowerShell 等主流 Shell，并可在 macOS、Windows 和 Linux 平台上运行。Warp 的核心理念是通过自然语言交互和上下文感知能力，让开发人员能够直接通过提示词编写代码、调试问题和执行命令，从而显著提高开发效率。

## 核心功能

<Tabs>
  <Tab title="AI 代理模式">
    <p>Warp 的 Agent Mode 是核心功能，能够自动识别自然语言输入并调用 AI 完成任务。开发人员可以通过 <code>Ctrl+I</code>（Windows）或输入 <code>* </code>（星号+空格）进入代理模式，然后用自然语言描述任务。</p>

    <p><strong>业务场景支持：</strong></p>
    <ul>
      <li>当开发人员不确定命令语法时，可直接输入"列出当前目录下所有大于 10MB 的文件"，AI 会生成并解释相应的命令</li>
      <li>代码调试时，可以将错误输出附加到 AI 查询中，系统会自动分析并建议修复方案</li>
      <li>AI 支持多轮对话和自我纠错，如果建议的命令失败，会自动请求执行新命令直到完成任务</li>
    </ul>
  </Tab>

  <Tab title="代码编辑">
    <p>Warp 可以索引 Git 代码库并生成上下文感知的代码编辑建议。当进入 Git 仓库时，Warp 会初始化代码库索引并生成 <code>WARP.md</code> 文件用于存储项目级规则。</p>

    <p><strong>业务场景支持：</strong></p>
    <ul>
      <li><strong>单文件修改</strong>：例如"将此文件中的所有 <code>var</code> 替换为 <code>let</code>"</li>
      <li><strong>批量修改</strong>：例如"为该目录下所有 <code>.py</code> 文件添加文件头"</li>
      <li><strong>基于错误输出的修复</strong>：当 TypeScript 编译失败时，可直接让 AI 根据错误信息修复代码</li>
    </ul>

    <p>开发人员还可以通过 Rules 功能定义全局或项目级编码标准，让 AI 生成符合团队规范的代码。</p>
  </Tab>

  <Tab title="Blocks 编辑器">
    <p>Warp 将命令输入和输出组织为 Blocks，每个 Block 都是独立单元，支持快速导航、复制和搜索。输入编辑器采用类似 IDE 的体验，支持多行编辑、光标跳转和智能选择。</p>

    <p><strong>业务场景支持：</strong></p>
    <ul>
      <li>在查看大量日志时，Sticky Command Header 会固定显示当前 Block 的命令，方便快速定位</li>
      <li>开发人员可以双击智能选择 URL、文件路径或 IP 地址，无需手动拖拽</li>
      <li>使用 <code>Ctrl+Alt</code> 加鼠标拖拽可进行矩形选择，适合复制结构化日志或列数据</li>
    </ul>
  </Tab>

  <Tab title="Workflows">
    <p>Workflows 是参数化的可执行命令模板，采用 YAML 格式定义，可通过 Command Search（<code>Ctrl+R</code>）快速搜索和执行。</p>

    <p><strong>业务场景支持：</strong></p>
    <ul>
      <li>团队可以将常用的部署脚本保存为 Workflow，新成员无需记忆复杂命令即可执行</li>
      <li>支持参数化配置，同一个 Workflow 可以适配不同环境和场景</li>
      <li>通过 Warp Drive 共享这些对象，确保团队使用统一的操作流程</li>
    </ul>
  </Tab>

  <Tab title="Notebooks">
    <p>Notebooks 是可运行的文档，包含 Markdown 文本和可执行的 Shell 代码块。</p>

    <p><strong>业务场景支持：</strong></p>
    <ul>
      <li>Notebooks 可作为运维手册，包含操作说明和可直接运行的命令，实现文档和实操的统一</li>
      <li>支持导出为 Markdown 格式，便于版本控制和知识传承</li>
      <li>团队可以通过 Warp Drive 共享 Notebooks，标准化操作流程</li>
    </ul>
  </Tab>

  <Tab title="Warp Drive">
    <p>Warp Drive 是一个侧边栏工作区，用于存储和管理 Workflows、Notebooks、Prompts 和环境变量。开发人员可以通过 <code>Ctrl+Shift+.</code>（Windows）打开该面板。</p>

    <p><strong>业务场景支持：</strong></p>
    <ul>
      <li>个人工作区用于保存私有命令和脚本，团队工作区则用于共享协作内容</li>
      <li>支持将对象组织到文件夹中，便于分类管理</li>
      <li>所有对象支持导入导出（Workflows 为 YAML，Notebooks 为 Markdown，环境变量为 .env）</li>
    </ul>
  </Tab>
</Tabs>

## Windows 11 安装指南

### 系统要求

在开始安装前，请确保系统满足以下要求：

- Windows 10 版本 1809（内部版本 17763）或更高版本
- Windows Server 2019（内部版本 17763）或 Windows Server 2022（内部版本 20348）或更高版本
- 支持 Windows Pseudo Console (ConPTY) 功能

### 安装步骤

<Steps>
  <Step title="下载安装程序">
    <p>访问 Warp 官方下载页面（<a href="https://app.warp.dev/download">https://app.warp.dev/download</a>），下载适用于 Windows 的安装程序。Warp 支持 x86-64 和 ARM64 架构。</p>
    
    <p>或者使用 WinGet 包管理器安装：</p>
    ```
    winget install Warp.Warp
    ```
  </Step>

  <Step title="执行安装">
    <p>双击下载的安装程序并按照提示完成安装。安装完成后，可在开始菜单中找到 Warp。</p>
  </Step>

  <Step title="首次启动配置">
    <ul>
      <li><strong>登录（可选）</strong>：首次启动时可选择通过 Google 或 GitHub 账号注册，也可跳过此步骤。Warp 仅获取关联的邮箱地址，不会访问其他数据</li>
      <li><strong>选择默认 Shell</strong>：Warp 会尝试加载系统登录 Shell，目前支持 bash、fish、zsh 和 PowerShell。可在 Settings > Features > Session 中修改默认 Shell</li>
      <li><strong>导入设置（可选）</strong>：如果从其他终端（如 Windows Terminal）迁移，可导入键盘快捷键和配色主题</li>
    </ul>
  </Step>

  <Step title="验证安装">
    <p>首次打开时需要网络连接，之后可离线使用，但 AI 和协作功能需要联网。安装完成后，Warp 会自动配置更新机制，确保始终使用最新版本。</p>
  </Step>
</Steps>

## 快速入门场景

<AccordionGroup>
  <Accordion title="使用 AI 生成和执行命令">
    <Steps>
      <Step title="启动 Agent Mode">
        <p>打开 Warp 终端窗口，按 <code>Ctrl+I</code> 或输入 <code>* </code>（星号+空格）进入智能代理模式。</p>
      </Step>
      
      <Step title="描述任务需求">
        <p>用自然语言清楚地描述你的任务需求，例如：</p>
        <p><em>"找出当前目录下最大的 5 个文件并按大小排序"</em></p>
      </Step>
      
      <Step title="生成并执行命令">
        <p>按 <code>Enter</code> 让 AI 分析需求并生成相应命令，命令会自动填充到输入框中。</p>
      </Step>
      
      <Step title="执行或退出">
        <p>再次按 <code>Enter</code> 执行生成的命令，或按 <code>Esc</code> 退出 Agent Mode 进行手动编辑。</p>
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="快速搜索和执行历史命令">
    <Steps>
      <Step title="打开搜索面板">
        <p>按 <code>Ctrl+R</code> 打开 Command Search 面板，这里汇集了命令历史、Workflows 和 Notebooks。</p>
      </Step>
      
      <Step title="模糊搜索命令">
        <p>输入关键词进行智能模糊搜索，系统会自动匹配相关的历史命令和工作流。</p>
      </Step>
      
      <Step title="选择目标命令">
        <p>使用 <code>↑</code> / <code>↓</code> 方向键浏览搜索结果，找到需要的命令。</p>
      </Step>
      
      <Step title="插入并执行">
        <p>按 <code>Enter</code> 将选中的命令插入到输入编辑器，根据需要修改参数后执行。</p>
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="创建和执行 Workflow">
    <Steps>
      <Step title="创建新工作流">
        <p>在 Warp Drive 面板（按 <code>Ctrl+Shift+.</code> 打开）中点击 "New Workflow" 开始创建。</p>
      </Step>
      
      <Step title="定义工作流配置">
        <p>使用 YAML 格式定义工作流名称和命令：</p>
        
        ```yaml path=null start=null
        name: 部署到生产环境
        command: ssh user@prod-server 'cd /app && git pull && ./deploy.sh'
        ```
      </Step>
      
      <Step title="保存并使用">
        <p>保存工作流后，通过 <code>Ctrl+R</code> 搜索该工作流名称即可快速执行。</p>
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="使用 Notebooks 作为操作手册">
    <Steps>
      <Step title="创建新笔记本">
        <p>在 Warp Drive 中点击 "New Notebook" 创建一个可执行的文档。</p>
      </Step>
      
      <Step title="编写文档内容">
        <p>结合 Markdown 说明和可执行代码块，创建完整的操作手册或技术文档。</p>
      </Step>
      
      <Step title="交互式执行">
        <p>保存后，点击 Notebook 中代码块旁的运行图标，命令会自动插入到终端输入框等待执行。</p>
      </Step>
      
      <Step title="执行命令">
        <p>按 <code>Enter</code> 执行插入的命令，实现文档与操作的无缝结合。</p>
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="调试代码并自动修复错误">
    <Steps>
      <Step title="执行代码测试">
        <p>在 Git 仓库目录中运行代码编译、测试或构建命令。</p>
      </Step>
      
      <Step title="触发 AI 分析">
        <p>当出现错误时，点击输出 Block 旁的 AI 图标（sparkles icon）启动智能分析。</p>
      </Step>
      
      <Step title="附加错误上下文">
        <p>选择 "Attach blocks to AI query" 将完整的错误信息附加到 AI 查询中。</p>
      </Step>
      
      <Step title="获取修复方案">
        <p>输入自然语言描述，例如：<em>"修复这个 TypeScript 编译错误"</em>，AI 会分析并提供具体的修复代码。</p>
      </Step>
      
      <Step title="应用修复">
        <p>按照 AI 的建议应用代码修改，完成错误修复。</p>
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="多窗格同步执行命令">
    <Steps>
      <Step title="分割终端窗格">
        <p>按 <code>Ctrl+Shift+D</code> 水平或垂直分割窗格，创建多个独立的终端会话。</p>
      </Step>
      
      <Step title="打开目标会话">
        <p>在各个窗格中分别打开需要同时操作的不同项目或服务器会话。</p>
      </Step>
      
      <Step title="启用同步输入">
        <p>在 Command Palette 中搜索并启用 "Synchronized Inputs" 功能。</p>
      </Step>
      
      <Step title="同步执行命令">
        <p>在任意一个窗格中输入命令，所有启用同步的窗格都会同时执行相同的命令。</p>
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

## 其他实用功能

<Columns>
  <Card title="Command Search 和命令纠错" icon="search">
    <p>Command Search（<code>Ctrl+R</code>）整合了命令历史、Workflows、Notebooks 和 Agent Mode 历史的统一搜索。Command Corrections 功能基于开源项目构建，可自动检测命令拼写错误、缺失参数并提供修复建议。</p>

    <p><strong>典型场景：</strong></p>
    <ul>

      <li>执行 <code>git push</code> 失败时，会提示添加 <code>--set-upstream</code> 参数</li>
      <li>模糊搜索功能让开发人员可以快速找到历史执行过的复杂命令</li>
    </ul>
  </Card>

    <p><strong>主要优势：</strong></p>
    <ul>
      <li>团队工作区实现跨团队协作</li>
      <li>支持多种格式导入导出</li>
      <li>统一的操作规范和最佳实践</li>
    </ul>
  </Card>
</Columns>
