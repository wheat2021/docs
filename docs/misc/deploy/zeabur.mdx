## 🧭 一、Zeabur 是什么？

**Zeabur** 是一个新一代的云端部署平台，主打 **“一键部署、自动扩容、多语言支持”**，主要面向开发者、初创团队和独立开发者。
它的定位类似 **Vercel + Docker + 微服务编排平台** 的结合体。

官网：🌐 [https://zeabur.com](https://zeabur.com)

核心理念是：

> “让部署变得像写代码一样简单。”

---

## 🚀 二、主要功能特性

| 功能                      | 说明                                                                   |
| ----------------------- | -------------------------------------------------------------------- |
| 🌍 **多语言支持**            | 自动识别并支持 Node.js、Python、Go、Rust、Deno、PHP、.NET、Java 等语言项目。             |
| 🐳 **Docker 支持**        | 可以直接部署自定义 Dockerfile，支持 Docker Compose 模式的部分功能（见下文）。                 |
| 🧩 **Service-Based 架构** | 每个应用由多个「服务（Service）」组成，支持 Web、Worker、Database、Cron 等类型。              |
| ☁️ **一键部署**             | 直接从 GitHub / GitLab / Docker 镜像导入，一键部署。                              |
| 🪄 **环境变量与密钥管理**        | 提供安全的 Secret 管理，支持在部署中自动注入环境变量。                                      |
| 📦 **数据库即服务**           | 提供 PostgreSQL、MongoDB、Redis、MySQL、PlanetScale、Neon、Upstash 等托管数据库集成。 |
| ⚙️ **自动扩容与监控**          | 根据流量自动扩容或缩容，无需手动管理实例。                                                |
| 🔄 **持续部署（CI/CD）**      | 每次推送代码到 main 分支都会自动触发部署。                                             |
| 🧠 **智能推理与依赖分析**        | 自动识别应用框架（Next.js、FastAPI、Flask、NestJS、Remix、Astro、SvelteKit、Nuxt 等）。 |
| 🌐 **域名与 HTTPS 管理**     | 支持自定义域名，自动签发 SSL 证书。                                                 |

## 🧱 三、是否支持 Docker Compose？

**部分支持** ✅

目前 Zeabur 支持 **单个服务使用 Dockerfile 部署**，但尚不支持直接上传完整的 `docker-compose.yml` 文件来编排多个容器。

不过你可以通过 **Zeabur Dashboard** 创建多个服务（Service）来模拟 Compose 中的多个容器，然后：

* 通过 **Environment Variables** 或 **Internal Networking** 让它们互相通信；
* 每个服务可以独立配置镜像、启动命令、端口等。

> 🔧 举例：
>
> * 一个 `web` 服务使用 `Dockerfile` 启动你的前端；
> * 一个 `db` 服务使用 Zeabur 提供的 PostgreSQL；
> * 一个 `worker` 服务跑后台任务。

未来官方计划提供类似 `docker-compose` 的一键多服务部署功能（Roadmap 中已提到）。

## 🧠 六、适合使用 Zeabur 的场景

✅ 独立开发者想快速上线个人项目（网站、API、Bot）。
✅ 想部署 Dockerfile 项目但不想维护服务器。
✅ 小团队需要快速搭建 Demo、微服务系统。
✅ 想要支持多语言、数据库、后台任务的云环境。
✅ 想要比 Vercel 更灵活、但比 AWS 更简单的解决方案。

## 🪜 七、入门方式

1. 注册账号：[https://zeabur.com](https://zeabur.com)
2. 连接 GitHub 仓库或上传 Dockerfile
3. Zeabur 会自动识别框架并部署
4. （可选）添加数据库服务、配置环境变量
5. 自动生成域名并上线
