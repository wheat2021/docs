**Koyeb** 是一个现代的云原生应用托管平台（serverless platform for web apps and APIs），主打极简部署、自动扩展、全球分发与 DevOps 自动化。其定位类似 Vercel、Fly.io 与 Heroku 的结合体，但以容器（Docker）为基础而非特定运行时。

## 一句话总结

> Koyeb 是一个基于容器的全球无服务器平台，用最少的配置即可在全球范围内部署、扩展和管理应用。

## 部署方式

<Tabs>
  <Tab title="Git 仓库">
    在 Koyeb 控制台选择 GitHub/GitLab 仓库，平台将自动检测语言、构建与运行命令。

```bash
# 在控制台选择仓库后，后续 push 将自动触发部署
```
  </Tab>
  <Tab title="Docker 镜像">
    若已有镜像，可直接指定镜像进行部署。

```bash
koyeb app create my-app --image ghcr.io/username/image:latest
```
  </Tab>
  <Tab title="Dockerfile">
    在仓库根目录提供 Dockerfile，平台将基于该文件构建并运行。

```dockerfile
FROM node:18
WORKDIR /app
COPY . .
RUN npm install
CMD ["npm", "start"]
```
  </Tab>
</Tabs>

## 快速开始

<Steps>
  <Step title="注册并登录">
    访问 https://www.koyeb.com 创建账号并登录控制台。
  </Step>
  <Step title="创建应用">
    在控制台创建应用，选择部署来源（Git 仓库、容器镜像或 Dockerfile）。
  </Step>
  <Step title="配置环境">
    按需设置环境变量与密钥，选择地区与实例规格。
  </Step>
  <Step title="触发部署">
    首次导入后自动构建并部署；后续对主分支的提交将触发持续部署。
  </Step>
  <Step title="域名与上线">
    使用系统提供的域名或绑定自定义域名，完成上线与验证。
  </Step>
</Steps>

## 与其它平台比较

| 平台                 | 部署类型              | 支持语言 | 定价模式         | 全球分发 | 亮点                   |
| -------------------- | --------------------- | -------- | ---------------- | -------- | ---------------------- |
| Koyeb                | Docker 容器           | 任意     | 免费层 + 按用量  | 是       | 全自动化、全球加速、零冷启动 |
| Vercel               | 框架级（对 Next.js 友好） | JS/TS   | 免费层 + 增值     | 是       | 极致前端体验           |
| Fly.io               | Docker 容器           | 任意     | 按实例           | 是       | 边缘部署、数据库托管   |
| Render               | 自动化容器            | 常见语言 | 免费层 + 固定计费 | 否       | 简洁易用，适合中小项目 |
| Heroku               | Buildpack             | 常见语言 | 固定计费         | 否       | 成熟生态但相对老旧     |
| Cloud Run (GCP)      | Docker 容器           | 任意     | 按请求计费       | 是       | 企业级可靠但配置较复杂 |
